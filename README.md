# Aegis Defense Tool: 開発の経緯と設計思想

## 1. 開発の経緯
私は大学院でネットワーク侵入検知システム（NIDS）の研究を行う傍ら、デジタルイラストの制作を続けています。昨今の生成AIの普及により、公開した作品が「意図しない学習データ（Data Misuse）」として消費される現状を、一人のクリエイターとして、またセキュリティを学ぶ学生として無視できないと感じたのが開発のきっかけです。

本プロジェクトは、単なる画像編集ツールの作成ではなく、データの機密性と完全性を守る「データガバナンス」の観点から、エンジニアとしてどのように権利保護を実現できるかという実証コードとしての側面を持っています。

## 2. 設計思想：非対称的な防御の実装
AIの学習は膨大なリソースを必要としますが、データ側に「微小な摂動（Perturbation）」を加えることで、その学習効率を低下させることが可能です。本ツールでは、学習データとしての価値を低下させ、学習コストを増大させる「非対称的な防御」をコンセプトに、以下の3層構造で構築しました。

* **視覚的レイヤー**: 著作権表示による無断転載の抑止。
* **アルゴリズムレイヤー (Mist)**: `os.urandom` によるランダムノイズを重ね合わせ、特徴量抽出を阻害。
* **データレイヤー (Poisoning)**: PNGメタデータにAI学習用のネガティブタグを注入し、モデルのラベル精度を低下。

## 3. 技術的な工夫

### 相対座標による配置管理
GUI上でのプレビューと最終的な高解像度出力で整合性を保つため、キャンバス上の座標を `0.0` 〜 `1.0` の相対座標で管理しています。 これにより、解像度が異なる画像に対しても、常に意図した位置に正確にウォーターマークを配置できるロジックを構築しました。

### 非破壊的な処理パイプライン
高解像度イラストの処理負荷を抑えるため、プレビュー表示時にはリサイズしたキャッシュ画像に対してのみ動的にフィルタを適用し、保存時にのみフル解像度へ反映する設計にしています。

### 実行環境に依存しないデータ管理
作業の連続性を確保するため、スライダー等の変更を検知した瞬間に設定ファイルを書き出す「イベント駆動型」の保存処理を実装しています。 また、スクリプトの実行パス（`BASE_DIR`）を基準とした絶対パス解決を行うことで、どの階層からプログラムを呼び出しても設定が正しく保持されるようにしました。

### 検証機能の統合
画像に含まれるメタデータを解析し、特定の汚染タグ（`bad anatomy` 等）を検知して `[!! POISON !!]` ラベルを表示する機能を組み込みました。 これにより、ユーザー自身が自分の画像が正しく保護されているかを即座に検査できる仕組みを提供しています。

## 4. 展望
今後は、自身の研究テーマである「モデル劣化スキーム」の知見をさらに応用し、特定のアーキテクチャに対してより特化した阻害効果を持つ動的なノイズ生成機能の実装を目指しています。
